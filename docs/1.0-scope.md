# NCP 1.0 candidate scope

> **Status:** `1.0.0-rc.1`, wire `1.0`, compact proto hash
> `163acc57d8a62b66`; unreleased and release-blocked. The exact normative source
> set and complete SHA-256 digest are generated in
> [`../contract/manifest.v1.json`](../contract/manifest.v1.json).

NCP 1.0 is the bounded canonical-JSON contract named in
`contract/surface.v1.json`. The machine-readable contract registries are normative;
`proto/ncp.proto` defines message field numbers and shapes; JSON Schemas define the
runtime projection; the prose specification explains those rules; the mandatory
corpus makes each stable rule executable. Earlier items in that order win if prose
or examples disagree.

Stable status is explicit. A message, capability, encoding, transport, package, or
extension not listed as `stable-1.0` is not stable by implication. Unknown and
experimental capabilities cannot be negotiated as stable.

`BulkObservation` and bare `NCPB` are excluded from the 1.0 transport surface.
The bounded `BulkBlock` codec remains available for local/offline experiments. NCP
1.0 does not ship protobuf as a runtime encoding, gRPC as a stable binding,
delegation/proxy authority, physical safety certification, a universal zero safe
action, or any paper-reproduction/calibrated-posterior claim.

The source is a release candidate until every required `pre_release_gates` entry in
`contract/release-gates.v1.json` is supported by evidence bound to one immutable
source and artifact set. Its `post_release_validations` run only after publication
and do not block the initial tag. The current Zenoh adapter also lacks callback-visible
authenticated peer identity and fails closed on `production-secure`; this
implementation prerequisite precedes the live mTLS gate. Local conformance cannot
satisfy independent-host, external consumer, signing, publication, or clean-room
reproduction gates. Engram's explicit native-1.0 migration is in progress but not
installed-artifact or live certified; the other five known consumers remain on
immutable wire 0.8. None is native 1.0 certified.

Local package archives do not establish public-registry ownership. The intended
crates.io `ncp-core` and PyPI `ncp` distribution names currently identify unrelated
projects, so ownership resolution or a coordinated package rename is a distinct
required pre-release gate.
